Fundamentos de SQL

Do Zero à Consultas Relacionais: Criação, Manipulação e Visões

Introdução

Básico Prático

O SQL (Structured Query Language) é a linguagem padrão para comunicação com bancos de dados relacionais. Nesta aula, exploraremos como estruturar bancos, criar entidades, relacionar dados e extrair informações de forma eficiente. Ao final, você dominará os comandos essenciais utilizados no mercado.

💡 Objetivo: Entender a sintaxe SQL, aplicar boas práticas de nomenclatura e executar consultas em ambientes reais (MySQL/PostgreSQL/SQLite).

1. Estruturação de Dados (DDL)

A linguagem de Definição de Dados (DDL) foca na criação e modificação da estrutura do banco. Os comandos mais utilizados são CREATE DATABASE, CREATE TABLE e definições de chaves.

-- Criação segura do banco de dados CREATE DATABASE IF NOT EXISTS `meu_banco`; USE `meu_banco`; -- Definição da entidade principal CREATE TABLE `Pessoa` ( `idpessoa` INT PRIMARY KEY, `nome_pessoa` VARCHAR(255), `cpf` CHAR(11), `nascimento` DATE );
⚠️ Atenção: Sempre utilize `USE nome_do_banco;` antes de executar comandos para garantir que você está operando no contexto correto e evitar alterações acidentais em outros bancos.
CREATE TABLE `Endereco` ( `idendereco` INT PRIMARY KEY, `idpessoa` INT, `rua` VARCHAR(255), `numero` INT, FOREIGN KEY (`idpessoa`) REFERENCES `Pessoa`(`idpessoa`) ); CREATE TABLE `Contatos` ( `idcontato` INT PRIMARY KEY, `idpessoa` INT, `celular` VARCHAR(20), `email` VARCHAR(100), FOREIGN KEY (`idpessoa`) REFERENCES `Pessoa`(`idpessoa`) );

As FOREIGN KEY garantem a integridade referencial. Elas vinculam registros da tabela filha a um registro válido na tabela pai.

2. Inserção de Dados (DML)

A Linguagem de Manipulação de Dados (DML) permite adicionar, atualizar ou remover registros. O comando INSERT INTO é fundamental para popular as tabelas.

-- Inserção múltipla em uma única execução INSERT INTO `Pessoa` (`idpessoa`, `nome_pessoa`, `cpf`, `nascimento`) VALUES (1, 'Ana Silva', '12345678901', '2000-01-01'), (2, 'João Souza', '23456789012', '2001-02-02'), (3, 'Maria Oliveira', '34567890123', '1998-05-15');
💡 Dica de Performance ▼

Inserções em lote (múltiplas linhas em um único INSERT) são significativamente mais rápidas do que executar um INSERT por linha, pois reduzem a sobrecarga de comunicação com o servidor.

4. Visões e Alterações (Avançado)

VIEW armazena uma consulta como uma tabela virtual. ALTER TABLE modifica estruturas existentes.

CREATE VIEW `Tele_lista_view` AS SELECT p.nome_pessoa, c.email, c.celular FROM Pessoa p JOIN Contatos c ON p.idpessoa = c.idpessoa; -- Consulta simplificada SELECT * FROM `Tele_lista_view`;
-- Remover índice antigo e adicionar novo ALTER TABLE `Contatos` DROP INDEX `idpessoa`, ADD INDEX `idnovo` (`idendereco`) USING BTREE; -- Renomear coluna e tipo ALTER TABLE `Contatos` CHANGE `idpess` `idendereco` INT; -- Atualizar registros UPDATE `Contatos` SET `idendereco` = NULL WHERE `idcontato` = 5;
