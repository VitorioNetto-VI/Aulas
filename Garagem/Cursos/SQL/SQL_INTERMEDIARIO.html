SQL Intermediário

Domine Joins, Views, Triggers e Subconsultas

Introdução

Intermediário Prático

Nesta aula, elevamos o nível de complexidade dos comandos SQL. Focaremos em manipulação avançada de dados, criação de estruturas de backup, automação via triggers e consultas que cruzam múltiplas tabelas com eficiência. O objetivo é preparar você para cenários reais de desenvolvimento e administração de bancos de dados.

Pré-requisito: Conhecimento básico de CREATE, INSERT, SELECT e WHERE.

1. Filtros e Subconsultas

O SQL permite buscas flexíveis e aninhamento de consultas para encontrar dados específicos. O operador LIKE facilita buscas parciais, enquanto subconsultas permitem filtrar resultados baseados em outra query.

-- Busca nomes que começam com 'João' SELECT * FROM Pessoa WHERE nome_pessoa LIKE 'João%'; -- Busca e-mails que contenham 'example' SELECT email FROM contatos WHERE email LIKE '%example%';
⚠️ Performance: O operador LIKE com % no início da string pode causar lentidão em tabelas muito grandes, pois impede o uso de índices. Use com cautela em produção.
-- Seleciona nomes de pessoas que moram no bairro 'Tropical ville' SELECT nome_pessoa FROM Pessoa WHERE idpessoa IN ( SELECT idpessoa FROM Endereco WHERE bairro = 'Tropical ville' );

A subconsulta interna retorna uma lista de IDs válidos, e a consulta externa filtra apenas as pessoas cujos IDs estão nessa lista.

2. Joins Avançados

Os JOINs são a espinha dorsal dos bancos relacionais. Entender a diferença entre INNER, LEFT e RIGHT é crucial para não perder dados ou gerar duplicatas.

-- Retorna apenas pessoas que possuem endereço cadastrado SELECT P.nome_pessoa, E.rua FROM Pessoa P INNER JOIN Endereco E ON P.idpessoa = E.idpessoa;

Registros que não têm correspondência na outra tabela são excluídos do resultado.

-- Retorna todas as pessoas, mesmo as sem endereço SELECT P.nome_pessoa, E.rua FROM Pessoa P LEFT JOIN Endereco E ON P.idpessoa = E.idpessoa;

Útil para identificar "gaps" nos dados, como clientes sem endereço cadastrado.

-- Retorna todos os endereços, mesmo os sem pessoa vinculada SELECT P.nome_pessoa, E.rua FROM Pessoa P RIGHT JOIN Endereco E ON P.idpessoa = E.idpessoa;

O inverso do LEFT JOIN: prioriza a tabela da direita.

3. Agrupamento e Filtragem Agregada

Utilizamos GROUP BY para consolidar dados e HAVING para filtrar esses grupos após a agregação (diferente do WHERE, que filtra linhas individuais).

-- Conta total de contatos por pessoa SELECT P.idpessoa, P.nome_pessoa, COUNT(C.idcontato) as total_contatos FROM Pessoa P LEFT JOIN Contatos C ON P.idpessoa = C.idpessoa GROUP BY P.idpessoa, P.nome_pessoa; -- Filtra apenas pessoas com mais de 1 endereço SELECT idpessoa, COUNT(*) as total_enderecos FROM Endereco GROUP BY idpessoa HAVING COUNT(*) > 1;
✅ Dica: Sempre inclua no GROUP BY todas as colunas que aparecem no SELECT e não são funções de agregação.

4. Views e Estruturas

Views simplificam consultas complexas e podem servir como camadas de segurança. Além disso, o ALTER TABLE permite modificar estruturas existentes.

-- Cria uma view para facilitar a leitura da lista telefônica CREATE VIEW Tele_lista_view AS SELECT pessoa.idpessoa, endereco.idendereco, contatos.idcontato FROM pessoa JOIN endereco ON pessoa.idpessoa = endereco.idpessoa JOIN contatos ON pessoa.idpessoa = contatos.idpessoa; -- Consulta a view como se fosse uma tabela SELECT * FROM Tele_lista_view; -- Adiciona coluna sobrenome na tabela Pessoa ALTER TABLE Pessoa ADD COLUMN sobrenome VARCHAR(255);

5. Triggers e Automação

Triggers são blocos de código que executam automaticamente antes ou depois de eventos (INSERT, UPDATE, DELETE). São ótimos para logs e integridade.

-- Antes de deletar, salva os dados na tabela de backup CREATE TRIGGER backup_pessoa BEFORE DELETE ON Pessoa FOR EACH ROW BEGIN INSERT INTO Pessoa_backup VALUES (OLD.idpessoa, OLD.nome_pessoa, OLD.cpf, OLD.nascimento); END;
-- Impede a exclusão de registros da tabela Pessoa CREATE TRIGGER alerta_exclusao BEFORE DELETE ON Pessoa FOR EACH ROW BEGIN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Não é possível excluir dados da tabela Pessoa'; END;

🎯 Desafio Prático

Avaliativo

Desenvolva um script SQL que:

  • Crie um banco `gestao_vendas`.
  • Tabelas: `Vendedores` (id, nome) e `Vendas` (id, id_vendedor, valor).
  • Insira 5 vendedores e 8 vendas.
  • Crie uma VIEW que mostre o nome do vendedor e o total vendido por ele.
  • Use um SELECT com HAVING para listar apenas vendedores que venderam mais de R$ 1000,00 no total.
  • Crie um TRIGGER que impeça a exclusão de um vendedor se ele tiver vendas registradas.
✅ Critérios: Uso correto de JOIN, GROUP BY, HAVING, VIEW e TRIGGER. Sintaxe limpa.
